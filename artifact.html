Project Manifest

NEW dashboard/index.html

Anti-slop dashboard — 10 JARVIS HUD UX patterns (clip-path chamfered cards, staggered panelIn animation, phosphor status dots, button hover glow, animated UEFI POST, CRT scanlines, holographic modal, canvas thermal PID ring, footer status bar, scrollbar polish). Voice command pipeline with SpeechRecognition API (11 commands). SSE EventSource client + WebSocket integration. Chart.js telemetry, IR heatmap, simulation controls.

NEW uefi_firmware.py

UEFI Boot ROM — POST (6 checkpoints), PLL config with frequency step (Q8.8), thermal fuse tables, CDC yield calibration, thermal wrap-up (8-step emergency shutdown). 17 readable registers at 0xF000_0000.

NEW jarvis_demo.html

Standalone JARVIS HUD demo — cinematic boot sequence, animated arc reactor ring (canvas rAF, multi-layer, pulse, tick marks, state-color mapping), 10 HUD panels, demo simulation (click ring or Space), typed chat with demo replies, B for boot replay, holographic welcome panel, mobile-responsive. Zero dependencies, no server needed.

NEW start_jarvis.bat

Windows double-click launcher — opens jarvis_demo.html in the default browser.

NEW dashboard/data/snapshot.json

Cached telemetry snapshot with UEFI register state, project config, regression results, VCD metadata, and physical model outputs.

Phase 6 — JARVIS Demo + Voice Pipeline
Created standalone JARVIS HUD demo + integrated voice commands into NeuroMesh

Phase 7 — SSE + WebSocket Hybrid
Replaced HTTP polling with real-time streaming endpoints
  • Added EventBus (publish/subscribe) to serve_dashboard.py with thread-safe queue management
  • Added GET /api/events SSE endpoint — bootstrap snapshot on connect, sim_start/seed_log/seed_result/sim_complete/sim_error events, 30s keepalive
  • Added /ws WebSocket endpoint — RFC 6455 handshake, bidirectional: voice_transcript (browser→server), simulation events (server→browser), ping/pong
  • run_regression() publishes events via EventBus in real-time (seed transitions, log lines)
  • Fixed 2 dead-code bugs in serve_dashboard.py (/api/status and /api/uefi/wrapup)
  • Read JARVIS ARCHITECTURE.md for protocol design inspiration

Voice Commands (Dashboard)

SAY "run simulation"

Triggers UCIe regression with current seed/cmd/update params

SAY "run thermal"

Executes thermal PID controller (reads limit/clk/load inputs)

SAY "run sweep"

Runs CDC yield frequency sweep via UEFI registers

SAY "boot"

Triggers UEFI POST with animated sequential checkpoints

SAY "show registers"

Opens holographic register dump modal (3D approach)

SAY "refresh"

Polls latest snapshot and re-renders all panels

SAY "shutdown"

Initiates 8-step thermal wrap-up emergency sequence

SAY "show credits"

Displays build credits in terminal

SAY "status"

Reports current system status (refreshes first)

SAY "help"

Lists all available voice commands in terminal
